SHOPIFY (WITH Ship os)
############################################################

OVERVIEW

Shopify is a hosted ecommerce platform where stores are managed through an admin dashboard. ShipOS integrates as an app inside Shopify to handle shipment creation, label printing, tracking, and automation.
Pricing tiers: Basic, Shopify, Advanced, Plus (higher tiers unlock advanced shipping and automation features).
Best for: Businesses needing fast setup with integrated shipping automation via ShipOS.

KEY CONCEPTS

Orders and ShipOS Flow
Orders are created in Shopify and must be manually or automatically sent to ShipOS. Only open orders (not older than 60 days) can be sent for shipment.

Shipment Tab (ShipOS)
Central workspace where all shipments are created, viewed, filtered, and managed. Includes actions like tracking, printing labels, canceling shipments, and adding notes.

Shipping Company
A courier selected from dropdown during shipment creation. Must be configured in Settings with a valid Client Code.

Delivery Types
Regular Delivery (standard), Collecting (pickup), Double (priority delivery). Selected during shipment creation.

Collection Points
Pickup locations configured using Google Maps API. Customers or system can select nearest location based on address.

Label System
Supports A4 (8 labels per page) and Thermal printer (single label rolls). Label settings control whether product details and variations are printed.

Automatic Shipping
Feature that creates shipments automatically when order reaches specific status. Requires license selection and mapping.

Tracker Page
Customizable tracking page for customers. Displays shipment status, courier info, and optionally order items and contact details.

SMS System
Automated SMS notifications triggered by shipping status or method using configured templates.

WHAT IS SHIPOS?
============================================================

Overview

ShipOS is a shipping management platform built specifically for online stores in Israel. It lets you manage your entire shipping process from one place -- from creating a shipment to delivering it to your customer's door.

Who is ShipOS For?

ShipOS is for any online store owner in Israel, whether you ship a few packages a day or hundreds. If you sell online and need to ship products, ShipOS is designed for you.

Supported Platforms

ShipOS works with the most popular e-commerce platforms:
- WooCommerce -- A dedicated plugin for WordPress stores with full integration
- Wix -- An app for Wix online stores
- Shopify -- An app for Shopify online stores
- Direct Connection -- Open connection for any custom system

What Can ShipOS Do?

CREATE SHIPMENTS
- Create a shipment with one click directly from an order
- Create shipments in bulk -- select multiple orders and ship them all at once
- Create a shipment manually without an existing order
- Automatic shipping -- ShipOS creates a shipment automatically when a new order comes in

PRINT SHIPPING LABELS
- Print a single label or labels for multiple orders at once
- Works with thermal label printers and regular A4 printers
- Option to print product details on the label
- Connect to PrintNode for remote printing

TRACK SHIPMENTS
- Real-time tracking of every shipment
- A dedicated tracking page for your customers
- Automatic order status updates in your store

PICKUP POINTS
- Show pickup locations at checkout so customers can collect their package nearby
- Support for list view, map view, or both
- Automatic syncing of pickup locations from your shipping company

RETURNS AND EXCHANGES
- A return portal where customers can request returns on their own
- Set your own return policy
- Manage all return requests from one dashboard

Shipping Companies

ShipOS connects with 15 Israeli shipping companies, including Chita, HFD, Orian, Cargo, LionWheel, GetPackage, Wolt Drive, and more. For the full list, see Supported Shipping Companies.

How to Get Started

1. Install the plugin/app -- Add ShipOS to your store
2. Create an account -- Sign up for a ShipOS account
3. Connect a shipping company -- Enter the credentials you received from your shipping company
4. Set up your preferences -- Walk through the setup wizard
5. Start shipping -- Create shipments directly from your store

Support
- Website: shipos.co.il
- App: app.shipos.co.il
- Support email: support@shipos.co.il

============================================================
FREQUENTLY ASKED QUESTIONS
============================================================

How much does ShipOS cost?
ShipOS works on a subscription model with different plans based on shipment volume and features. Visit shipos.co.il or contact sales for pricing.

How do I connect a shipping company?
1. Open ShipOS dashboard
2. Click Licenses or the Setup Wizard
3. Pick your shipping company
4. Enter credentials (Client Code / Username / Password)
5. Enter pickup address
6. Save and verify connection

What do I do if a shipment fails?
Common causes:
1. Wrong/expired credentials
2. Incomplete address (need city, street, house number)
3. Invalid phone number (must be Israeli format)
4. Expired license
5. Connection issue with ShipOS server

How do I print a label?
Single: Open order > In ShipOS box click Print Label > PDF downloads
Bulk: Select orders in list > Bulk Actions > Print Labels > one PDF for all

You can choose thermal labels or A4 in ShipOS settings.

What are pickup points?
Pickup points are physical locations (convenience stores, post offices, parcel lockers) where customers can collect their package instead of getting it delivered home.

How does it work?
1. Customer places order
2. At checkout, picks nearby pickup location
3. Package delivered to that location
4. Customer notified when ready
5. Customer picks it up

How to enable pickup points:
1. Make sure shipping company supports them
2. ShipOS settings > Pickup Point Settings
3. Enable Pickup Points
4. Sync from shipping company
5. Create "ShipOS Shipping" method in shipping zones

How do I set up automatic shipping?
1. ShipOS dashboard > Settings > Shipping
2. Turn on Automatic Shipping
3. Set: default company, status trigger, delivery type, package count
4. Important: it creates a shipment for EVERY order at that status — verify settings before enabling

What if there's a connection error?
1. Check internet
2. Refresh page
3. Clear cache
Firewall/Cloudflare: allow ShipOS server IPs:
  IPv4: 5.75.225.51
  IPv6: 2a01:4f8:c012:c6d4::1
WooCommerce store keys: WooCommerce > Settings > Advanced > check ShipOS connection keys; click Reauthorize if missing/expired.

Does ShipOS work with Hebrew-language stores?
Yes. Designed for the Israeli market: full Hebrew interface, RTL layout, Hebrew addresses, Israeli phone numbers.

Can I connect more than one shipping company?
Yes. Multiple connections supported. Each is a separate "license". Choose per-order or set a default for automatic.

Does ShipOS support international shipping?
Yes via ShipUPS (UPS network). Other companies are Israeli domestic.

Can I use different shipping companies for different products?
Yes. Set up shipping methods per shipping zone in WooCommerce, assign different companies to each.
Example zones:
- Tel Aviv: Wolt Drive (same-day)
- Rest of Israel: Zig Zag (standard)
- International: ShipUPS

Do I need a Google Maps API key?
Only if you want a MAP view of pickup points at checkout. List view doesn't need it.
Setup: Google Cloud Console > enable Maps JavaScript API > create key > paste in ShipOS pickup point settings.

Can I customize the tracking page?
Yes. Logo, brand colors, contact info, multiple page designs.

Is there a mobile app?
No separate mobile app for store owners. Dashboard works in any browser. Customer tracking page is mobile-friendly.

What label printers are supported?
- Thermal: Zebra, DYMO, TSC etc. (use "Thermal" format)
- Regular: any A4 inkjet/laser (use "A4" format)


============================================================
TROUBLESHOOTING
============================================================

"CANNOT CONNECT TO SHIPOS SERVER"
1. Firewall: allow IPv4 5.75.225.51 and IPv6 2a01:4f8:c012:c6d4::1
2. Cloudflare: Security > WAF > add exception for ShipOS server addresses
3. Verify HTTPS certificate is valid (not expired)
4. Hosting: ask provider to allow outgoing connections to api.shipos.co.il

"CONSUMER SECRET IS INVALID"
1. WordPress admin > WooCommerce > Settings > Advanced > look for ShipOS keys
2. If missing/expired:
   - Open ShipOS dashboard > click Reauthorize
3. If still failing:
   - Delete ShipOS keys in WooCommerce
   - Re-authorize from ShipOS dashboard
Important: only delete ShipOS keys, not other services'.

"ERROR CREATING SHIPMENT"
1. Verify shipping company credentials (try logging in to their system with the same creds)
2. Address must include city + street + house number, no unusual characters
3. Phone: 10 digits, starts with 05 (mobile) or valid area code, no spaces/special chars
4. Pickup (sender) address set up correctly
5. Delivery type matches what shipping company supports

"GOOGLE MAPS KEY REQUIRED"
1. Google Cloud Console > new project
2. Enable Maps JavaScript + Places
3. Create key, restrict to your domain
4. ShipOS settings > Pickup Point Settings > paste key
Tip: choose "List" display to avoid needing a key.

SECURITY CERTIFICATE ISSUES
1. Test cert at SSL Labs
2. Ask hosting provider to update certs / enable modern TLS

PLUGIN CONFLICTS (WOOCOMMERCE)
1. Deactivate all plugins except WooCommerce + ShipOS
2. Reactivate one by one to find conflict
Common offenders: cache plugins, security plugins (Wordfence, iThemes Security), other shipping plugins, checkout customization plugins.
3. Try a default theme (Twenty Twenty-Four)

CLOUDFLARE BLOCKING SHIPOS
Cloudflare dashboard > Security > WAF > Custom rules > new rule:
  Name: Allow ShipOS
  Condition: IP Source equals 5.75.225.51
  Action: Allow
Add another for IPv6 2a01:4f8:c012:c6d4::1.
Watch for "Super Bot Fight Mode" blocking legitimate services.

ORDERS AREN'T SYNCING
1. WooCommerce > Settings > Advanced > Webhooks: ensure ShipOS notifications Active
2. ShipOS dashboard > click Sync Orders for manual sync
3. Verify ShipOS has read/write permissions on store data

LABEL ISN'T PRINTING CORRECTLY
1. Settings > Label > pick correct type (thermal vs A4)
2. Check rotation, try Fit to page, verify printer paper size
3. Thermal: roll width usually 10cm, check resolution

PLUGIN WON'T INSTALL
- WordPress 5.0+, WooCommerce 5.0+, PHP 7.4+
- Folder wp-content/plugins needs 755 permissions
- Try manual install: download ZIP, extract to wp-content/plugins/, activate

Provide: site URL, problem description, full error message, plugin/WC/WP versions, shipping company, order number.

CREATING MANUAL SHIPMENTS
============================================================

A manual shipment lets you create a delivery without an existing order. Use cases:
- Phone orders
- Replacements for damaged/lost items
- Gifts outside normal flow
- Internal transfers
- Product samples

How to create:
1. ShipOS dashboard from store admin
2. Click Manual Shipment / Create New Shipment
3. Fill recipient details:
   - Recipient name (required) - full name
   - Phone (required) - mobile, 10 digits starting 05
   - City (required)
   - Street (required)
   - House number (required)
   - Apartment / Floor (optional)
   - Email (optional, for tracking updates)
4. Choose shipping company
5. Select delivery type (regular, two-way etc.)
6. Set number of packages
7. Add courier notes (optional)
8. Click Create Shipment

After creation:
- Tracking number generated
- Can print label
- Appears in shipment list
- Status tracked through delivery process

Differences vs regular shipments:
- Not linked to any store order
- Won't update any order status
- Won't appear on order page in WC/Shopify/Wix
- Will appear in ShipOS dashboard + reports

Available on all platforms: WooCommerce, Shopify, Wix.
ShipOS doesn't store an address book for manual shipments — keep frequent recipients somewhere handy.

CREATING SHIPMENTS (WOOCOMMERCE)
============================================================

Single shipment from order:
1. WP admin > WooCommerce > Orders
2. Click order
3. ShipOS box on the side
4. Configure:
   - Shipping company (auto if only one)
   - Delivery type: Standard (one-way) or Two-way (deliver + pickup something on return, e.g. cash)
   - Number of packages (default 1)
   - Notes for courier (optional)
5. Click Create Shipment
6. Tracking number shown

Bulk shipments:
1. WooCommerce > Orders
2. Check the boxes for orders to ship
3. Bulk Actions: "Create Shipments - ShipOS" > Apply
4. Uses default shipping company, standard delivery, 1 package each
Important: all selected orders need complete address + valid phone, otherwise individual ones fail.

Manual shipment:
1. ShipOS dashboard > Manual Shipment / Create New Shipment
2. Fill: name, phone, address, company, delivery type, packages, notes
3. Click Create Shipment
Manual shipments don't link to a WC order, only show in ShipOS dashboard.

Automatic shipping:
1. ShipOS dashboard > Settings > Shipping
2. Turn on Automatic Shipping
3. Set: default company, status trigger (e.g. "Processing"), delivery type, default package count

Use auto when:
- All orders go through same company
- Delivery type identical for all
- Want to save manual time
Don't use when:
- Different companies for different orders
- Mixed standard / two-way
- Variable package counts

Delivery types:
- Standard: one-way to customer
- Two-way: courier delivers + picks something up on return (cash, documents, returns) — not all companies support

Common errors:
"Invalid address" — needs city + street + house number, check spelling
"Invalid phone number" — 10 digits starting 05 (mobile) or valid area code
"License not active" — check expiry, sync licenses
"Server error" — retry, check connection, contact support

Single label from order page:

Single label from ShipOS dashboard:

4. One PDF with all labels downloads

Bulk labels (ShipOS dashboard):
1. ShipOS > Orders > select > Print Labels
Each label on its own page so you can print + cut.

Label types:
- Thermal: thermal label printers (Zebra, Dymo etc.), label-roll size, no cutting needed
- A4: regular printer, full A4 page, must cut after printing

Settings (ShipOS dashboard > Settings):
- Rotation: 0 / 90 / 180 degrees
- Fit to page: on/off
- Print products on label (with name, SKU, variations)

PRINTNODE (remote printing):
1. Account at PrintNode
2. Install PrintNode client on the printer's computer
3. ShipOS Settings > Label > paste PrintNode connection code
4. Pick printer
5. Save
After setup, "Print Label" sends straight to the printer (no PDF download).

============================================================
API OVERVIEW
============================================================

Base URL: https://app.shipos.co.il/api/

Authentication:
1. License-based (plugin endpoints): license_key + domain + token
   Example:
   curl -X POST https://app.shipos.co.il/api/shipping/open_new_order \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -d "license_key=YOUR_LICENSE_KEY" \
     -d "domain=yourstore.com" \
     -d "version=3.1.1"

2. Sanctum tokens (dashboard endpoints): Laravel Sanctum Bearer tokens
   curl -X GET https://app.shipos.co.il/api/licenses \
     -H "Authorization: Bearer YOUR_SANCTUM_TOKEN" \
     -H "Accept: application/json"
   Used by ShipOS dashboard, not external integrations.

Request format:
- Content-Type: application/x-www-form-urlencoded (or multipart/form-data) for POSTs
- Common params: license_key, domain, token, version

Response format (JSON):
Success:
  { "success": true, "data": { ... } }
Error:
  { "success": false, "message": "...", "errors": { ... } }

HTTP status codes:
200 success, 201 created, 400 bad request, 401 unauthorized, 403 forbidden,
404 not found, 422 validation, 429 rate limit, 500 server error

Rate limits:
- 60 req/min per license
- 120 req/min per IP
- 10 req/min for bulk operations
On 429: response includes retry_after seconds. Use exponential backoff.

Versioning: pass version=3.1.1 (use latest).

Best practices:
1. Always check `success` before reading `data`
2. Iterate `errors` object for validation feedback
3. Retry 5xx with exponential backoff
4. Log all errors
5. Never expose tokens client-side or in repos

  function getDistance(lat1, lon1, lat2, lon2) {
    const R = 6371;
    const dLat = (lat2-lat1)*Math.PI/180, dLon = (lon2-lon1)*Math.PI/180;
    const a = Math.sin(dLat/2)**2 +
              Math.cos(lat1*Math.PI/180)*Math.cos(lat2*Math.PI/180) *
              Math.sin(dLon/2)**2;

============================================================
WEBHOOKS
============================================================

Webhooks are HTTP callbacks notifying ShipOS when events happen in your WooCommerce store. The integration auto-syncs orders via three events:
- order_created
- order_updated
- order_deleted

URL pattern: https://app.shipos.co.il/api/wp/webhooks/{wp_site_id}/{event}
The wp_site_id distinguishes between different WC stores.

is_virtual field: the plugin filters line items to add this flag. Determines whether physical shipping is needed; if all items are virtual, no physical shipment is created. Verify the WC plugin remains active for this filter to work.

Payload follows the WooCommerce REST API Order schema with full order details (customer, items, shipping data, metadata). Delete events have minimal payload (just order ID).

Retry logic (WC built-in): up to 5 attempts with delays 1m, 5m, 30m, 1h, 6h. After 5 failures, webhooks are disabled and need manual re-enable.

Security: HMAC-SHA256 signatures in X-WC-Webhook-Signature header. Verify payload authenticity using the shared webhook secret.

============================================================

HFD: major Israeli delivery company. Newer/faster API integration.

Need:
- Connection code (HFD provides)
- Authentication token (HFD provides) — REQUIRED
- Collection address

Important: BOTH connection code AND auth token required. Either missing → connection fails.

Get credentials: contact HFD, mention you're using ShipOS, request connection code + auth token.

Connect:
1. WordPress admin > ShipOS > Settings > Licenses
2. Connect a new delivery company > pick "HFD"
3. Enter connection code + auth token
4. Enter collection address
5. Friendly name (e.g. "My HFD account")
6. Save

Features:
Regular Delivery: yes
Double Delivery: yes
Pickup Points: yes
COD (Govaina): yes
Express/Same-Day: no
Label Printing: yes

Troubleshooting:
- "Invalid connection code" / "Authentication failed": check both code and token
- "Token expired": ask HFD for new token, update in ShipOS
- Connection stuck: refresh + reconnect, contact support if persists

ShipOS מיועד לכל בעל חנות אונליין בישראל, בין אם אתה שולח כמה חבילות ביום ובין אם מאות.

פלטפורמות נתמכות:
- WooCommerce -- תוסף ייעודי לחנויות WordPress עם אינטגרציה מלאה
- Wix -- אפליקציה לחנויות אונליין של Wix
- Shopify -- אפליקציה לחנויות אונליין של Shopify
- חיבור ישיר -- חיבור פתוח לכל מערכת מותאמת אישית

מה ShipOS יכול לעשות?

יצירת משלוחים:
- יצירת משלוח בלחיצה אחת ישירות מהזמנה
- יצירת משלוחים בכמויות
- יצירת משלוח ידני ללא הזמנה קיימת
- משלוח אוטומטי

הדפסת מדבקות:

מעקב משלוחים:

האשף מלווה ב-5 שלבים אחרי חיבור החנות.

שלב 1: חיבור חברת משלוחים
- בחר מהרשימה הנפתחת
- הזן: קוד לקוח, שם משתמש, סיסמה
- כתובת איסוף: עיר, רחוב, מספר, מיקוד, איש קשר, טלפון

שלב 2: נקודות איסוף
- הפעל אם רוצים לאפשר ללקוחות לאסוף
- סנכרון אוטומטי מהחברה
- הגדר מחירים: רגיל / מוזל / סף הנחה
- בחר תצוגה: רשימה / מפה / שניהם
- מפה דורשת מפתח Google Maps

שלב 3: הגדרות משלוח
- הפעל יצירת משלוח אוטומטית?
- בחר סטטוס מפעיל (בעיבוד או מותאם)
- סוג מדבקה: תרמית או A4
- הדפסת מוצרים על מדבקה
- סנכרון הזמנות קיימות

שלב 4: איסוף עצמי
- הפעל איסוף מהמיקום שלך
- בחר שיטת משלוח תואמת (Local Pickup)

שלב 5: סיום
- סיכום ההגדרות
- אפשר ליצור משלוח ראשון, לבדוק קופה, לכוונן הגדרות, להוסיף עוד חברות

חזרה לאשף: לוח הבקרה > הגדרות (אפשר לעדכן הכל משם).

יצירת משלוחים (WooCommerce)
============================================================

משלוח בודד מהזמנה:
1. ניהול WP > WooCommerce > הזמנות
2. לחץ על ההזמנה
3. תיבת ShipOS בצד
4. הגדר:
   - חברת משלוחים (אם יש רק אחת -- אוטומטי)
   - סוג משלוח: רגיל (חד-כיווני) / דו-כיווני (משלוח + איסוף, למשל מזומן)
   - מספר חבילות (ברירת מחדל 1)
   - הערות לשליח (אופציונלי)
5. צור משלוח
6. מספר מעקב יוצג

משלוחים בכמויות:
1. WooCommerce > הזמנות
2. סמן הזמנות
3. פעולות מרובות: "צור משלוחים - ShipOS" > החל
4. משתמש בברירות מחדל
חשוב: כל ההזמנות חייבות כתובת מלאה + טלפון תקין, אחרת הן נכשלות פר-הזמנה.

משלוח ידני:
1. לוח הבקרה > משלוח ידני / צור משלוח חדש
2. מלא: שם, טלפון, כתובת, חברה, סוג, חבילות, הערות
3. צור משלוח
משלוחים ידניים לא מקושרים ל-WC, רק בלוח הבקרה.

משלוח אוטומטי:
1. לוח הבקרה > הגדרות > משלוחים
2. הפעל אוטומטי
3. הגדר: חברה, סטטוס מפעיל (למשל "בעיבוד"), סוג, חבילות
מתי כן: כל ההזמנות באותה חברה/סוג, חיסכון בזמן.
מתי לא: חברות/סוגים מעורבים, מספרי חבילות משתנים.

סוגי משלוח:
- רגיל: חד-כיווני
- דו-כיווני: השליח מוסר ואוסף משהו (מזומן/מסמכים/החזרה). לא כל החברות תומכות.

שגיאות נפוצות:
"כתובת לא תקינה" -- חסר עיר/רחוב/מספר, שגיאות כתיב
"מספר טלפון לא תקין" -- 10 ספרות, מתחיל 05 או קידומת תקינה
"רישיון לא פעיל" -- בדוק תפוגה, סנכרן רישיונות
"שגיאת שרת" -- נסה שוב, בדוק חיבור, פנה לתמיכה

הדפסת מדבקות (WooCommerce)
============================================================

מדבקה בודדת מדף הזמנה:
1. WC > הזמנות > לחץ הזמנה
2. תיבת ShipOS (חייב משלוח פעיל)
3. הדפסת מדבקה > PDF

מדבקה בודדת מלוח הבקרה:
1. ShipOS > הזמנות / משלוחים
2. סמל המדבקה > PDF

מדבקות מרובות (רשימת WC):
1. WC > הזמנות > סמן
2. פעולות מרובות: "הדפסת מדבקות - ShipOS"
3. PDF אחד יורד

מדבקות מרובות (לוח הבקרה):
1. ShipOS > הזמנות > סמן > הדפסת מדבקות
כל מדבקה בעמוד נפרד.

סוגי מדבקות:
- תרמית: מדפסות מדבקות תרמיות, ללא חיתוך
- A4: מדפסת רגילה, דף שלם, חיתוך אחרי

הגדרות (לוח הבקרה > הגדרות):
- סיבוב: 0 / 90 / 180
- התאם לעמוד: כן/לא
- הדפסת מוצרים: שם / מק"ט / וריאציות

PrintNode (הדפסה מרחוק):
1. חשבון ב-PrintNode
2. לקוח PrintNode על המחשב של המדפסת
3. ShipOS > הגדרות > מדבקות > קוד חיבור PrintNode
4. בחר מדפסת
5. שמור
"הדפסת מדבקה" שולחת ישירות -- ללא PDF.

בעיות נפוצות:
- ריקה: משלוח לא נוצר / חברה לא תומכת
- חתוכה: סיבוב, התאם לעמוד, גודל נייר
- ברקוד לא נסרק: איכות הדפסה, רזולוציה, מצב המדבקה
- לא יורד: חוסם פופ-אפים, דפדפן אחר


============================================================
נקודות איסוף (WooCommerce)
============================================================

מה זה?
מיקומים פיזיים -- מכולות, סניפי דואר, לוקרים, מרכזי איסוף.

יתרונות:
- זול יותר ממשלוח לבית
- גמיש ללקוח
- פחות משלוחים כושלים
- ידידותי לסביבה

הפעלה:
1. ניהול WP > WC > הגדרות > Shipos Delivery > הגדרות איסוף
2. סמן הפעל נקודות איסוף
3. שמור
דרישה: חברת משלוחים תומכת.

סנכרון:
- אוטומטי בהפעלה ראשונה
- ידני: כפתור רענון
טיפ: סנכרנו מדי פעם.

יצירת שיטת ShipOS Shipping:
1. WC > הגדרות > משלוחים
2. בחרו אזור (למשל ישראל)
3. הוסף שיטת משלוח > "ShipOS Shipping"
4. ערוך: שם (כפי שהלקוח רואה, למשל "איסוף מנקודה קרובה"), הפעל

מחירים:
- מחיר רגיל: ברירת מחדל
- מחיר מוזל: מתחת לסכום מסוים
- סכום מינימום להנחה
דוגמה: רגיל 25, מוזל 15, מינימום 200 ש"ח.
משלוח חינם: מחיר 0.

תצוגה:
- רשימה (מומלץ): לקוח בוחר עיר ואז מיקום. מהיר, ללא מפתח.
- מפה: Google Maps עם סיכות. ויזואלי, דורש מפתח, לאט יותר.
- שניהם: שילוב, אפשר להגדיר תצוגת ברירת מחדל.

מפתח Google Maps (רק מפה/שניהם):
1. Google Cloud Console > פרויקט
2. APIs & Services > Library > הפעילו Maps JavaScript + Places
3. APIs & Services > Credentials > מפתח חדש
4. הדביקו ב-ShipOS pickup
5. הגבילו: HTTP referrers > your-store.com/* + Maps JS + Places

חוויית הלקוח:
1. ממלא פרטים
2. אפשרות נקודת איסוף בשיטות משלוח
3. בוחר: רשימה / מפה
4. בוחר נקודה
5. כתובת מופיעה באישור

פתרון בעיות:
- לא מופיע בקופה: הפעלת? סנכרנת? יש שיטת ShipOS Shipping פעילה באזור?
- מפה לא נטענת: מפתח? שירותים? Console (F12); הגבלות.
- "נדרש מפתח Google Maps": עברו לרשימה או הזינו מפתח

מדריך הגדרות מלא (WooCommerce)
============================================================

ההגדרות מפוצלות לשני מקומות:
1. WooCommerce > הגדרות > Shipos Delivery
2. לוח הבקרה של ShipOS (iframe)

WooCommerce > הגדרות > Shipos Delivery:

לשונית כללי:
- חברת משלוחים ברירת מחדל (לחצו סנכרון רישיון לעדכון רשימה)
- חישוב משלוח חינם: אחרי הנחה / לפני הנחה
  דוגמה: סף 200, הזמנה 250 עם קופון 60 -- אחרי הנחה (190): לא זכאי, לפני הנחה (250): זכאי
- עדכון סטטוס הזמנה אוטומטי: כשהמשלוח נמסר
- סטטוס מקור (ברירת מחדל: בעיבוד)
- סטטוס יעד (ברירת מחדל: הושלם)
- עדכון לפי שיטת משלוח (למשל איסוף עצמי -> "מוכן לאיסוף")
- מצב בדיקה (למפתחים בלבד -- אל תפעילו בחנות פעילה!)

לשונית הגדרות איסוף:
- הפעל נקודות איסוף
- סנכרון מיקומים
- העדפת תצוגה: רשימה / מפה / שניהם
- מפתח Google Maps (אם מפה/שניהם)
- תצוגת ברירת מחדל (אם "שניהם")

לשוניות אינטגרציה: GetPackage, Wolt Drive, Zig Zag (כל אחת: הפעל + בחר חברה).

לוח הבקרה של ShipOS:

הגדרות משלוח:
- משלוח אוטומטי: חברה, סטטוס מפעיל, סוג, מספר חבילות
- סנכרון הזמנות: אילו סטטוסים לכלול

הגדרות מדבקה:
- סוג: תרמית / A4
- הדפסת מוצרים, מק"ט, וריאציות
- סיבוב: 0 / 90 / 180
- התאם לעמוד

שדות מותאמים:
- שדה טלפון, דירה, קומה, כניסה (אם תוסף קופה מוסיף שדות)

איסוף עצמי:
- הפעל
- שיטת משלוח לזיהוי

טיפים:
1. תמיד שמרו אחרי שינוי
2. סנכרנו רישיונות אחרי הוספת חברה ב-ShipOS
3. בדקו בקופה אחרי שינוי נקודות איסוף
4. התחילו עם ברירות המחדל ושפרו לפי הצורך


============================================================
ניהול הזמנות (WooCommerce)
============================================================

איך הזמנות מסתנכרנות:
ShipOS מקבל אוטומטית הזמנות חדשות, עדכונים, ביטולים, מחיקות.

סנכרון ידני:
1. לוח הבקרה > סנכרון הזמנות
טיפ: שימושי בחיבור חנות עם הזמנות קיימות.

אם סנכרון אוטומטי מפסיק:
1. WC > הגדרות > מתקדם > Webhooks
2. ודאו ש-ShipOS פעיל (Active). הפעילו אם Disabled
3. ודאו שאין חסימה של חומת אש / Cloudflare

צפייה בהזמנות:
לוח הבקרה > הזמנות. מציג: מספר הזמנה, תאריך, לקוח, טלפון, כתובת, סכום, סטטוס הזמנה, סטטוס משלוח, חברת משלוחים.

חיפוש: מספר הזמנה, שם, טלפון, מספר מעקב.
סינון: טווח תאריכים, סטטוס הזמנה/משלוח, חברה.

ייצוא ל-CSV: סננו ולחצו ייצוא. כולל את כל מידע ההזמנה. שימושי לדוחות וניתוח.

עדכון פרטים:
- טלפון: לחצו ועדכנו
- כתובת: לחצו ועדכנו
חשוב: עדכון לא משפיע על משלוח שכבר נוצר. אם צריך, בטלו וצרו חדש.

הערות:
- פנימיות (רק הצוות)
- לשליח (מועברות בעת יצירת משלוח)

פעולות:
- צור משלוח / הדפס מדבקה / צפה / עקוב
- מרובות: צור משלוחים, הדפס מדבקות, ייצוא


============================================================
החזרות והחלפות (WooCommerce)
============================================================

מה זה?
פורטל ייעודי שבו לקוחות מבקשים החזרות/החלפות בעצמם.

יתרונות:
- חוסך זמן לשירות לקוחות
- חוויית לקוח מובנית
- שקיפות
- ניהול מרכזי

הגדרת מדיניות:
לוח הבקרה > הגדרות > החזרות והחלפות.
- חלון החזרה (ימים לאחר מסירה, למשל 14)
- סיבות (פגום, מידה לא מתאימה, לא מרוצה, הזמנה בטעות)
- סוגי בקשות: החזרה / החלפה
- אילו מוצרים זכאים (אפשר להחריג קטגוריות)

תהליך הלקוח:
1. כניסה לפורטל (קישור מאתר/אימייל) -- מספר הזמנה + אימייל
2. אימות: קוד חד-פעמי לטלפון/אימייל
3. בחירת פריטים: מה להחזיר/להחליף, כמות, סיבה, הערה
4. שליחה -- אישור מתקבל

ניהול בקשות:
לוח הבקרה > החזרות והחלפות. מציג: מספר בקשה, מספר הזמנה, לקוח, פריטים, סיבה, סטטוס, תאריך.

טיפול:
- אישור: בדקו, ודאו עומד במדיניות, אשרו -- המערכת יוצרת משלוח החזרה (אם רלוונטי)
- דחייה: הוסיפו סיבה -- הלקוח מקבל הודעה

סטטוסים: חדשה / אושרה / נדחתה / במשלוח / התקבלה / הושלמה.

קישור פורטל ההחזרות:
https://app.shipos.co.il/returns/{store_slug}
מקומות מומלצים להוספה: תפריט/פוטר, דף מדיניות, אימייל אישור הזמנה, דף מעקב.

שאלות נפוצות:
- לקוח לא נכנס: בדקו מספר הזמנה, אימייל, חלון פתוח
- אין קוד אימות: טלפון/אימייל בהזמנה נכון? שירות SMS מוגדר?
- החזרה ידנית: כן, אפשר ליצור מלוח הבקרה

התקנת ShipOS על Wix
============================================================

דרישות:
- חנות Wix פעילה עם Wix eCommerce
- חשבון ShipOS (אפשר ליצור)
- חשבון חברת משלוחים אחת לפחות

שלב 1: התקנה מ-Wix App Market
1. Wix Dashboard > אפליקציות > בקר ב-App Market
2. חפש "ShipOS"
3. הוסף לאתר > בחר אתר > התקן

שלב 2: אישור הרשאות
- קריאה/ניהול הזמנות, מוצרים, מילוי הזמנות
- אשר

שלב 3: יצירת חשבון / התחברות
- חדש: צור חשבון > מלא פרטים > הירשם > אמת אימייל
- קיים: התחבר
טיפ: אותה התחברות עובדת בין WC/Wix/Shopify של ShipOS.

שלב 4: חיבור חברת משלוחים
1. ShipOS > הגדרות > רישיונות
2. הוסף רישיון חדש
3. בחר חברה > הזן פרטים > שמור
סטטוס Active. נסה משלוח בדיקה.

פתרון בעיות:
- אפליקציה לא מופיעה: רענן את לוח הבקרה > אפליקציות > ניהול
- אישור נכשל: נקה מטמון, דפדפן אחר
- לא יוצר חשבון: אימייל רשום, שכחתי סיסמה
- אימות רישיון נכשל: בדוק פרטים מול החברה


============================================================
יצירת משלוחים ב-Wix
============================================================

לפני: ShipOS מותקן ומחובר, רישיון פעיל, יש הזמנות.

משלוח בודד:
1. אפליקציית ShipOS מ-Wix Dashboard
2. הזמנות > לחץ הזמנה
3. צור משלוח
4. שדות (רובם מתמלאים אוטומטית): חברה, סוג, נמען, כתובת, טלפון, חבילות, הערות
5. בדוק > צור
6. מספר מעקב יוצר

משלוחים מרובים:
1. הזמנות > סמן בתיבות
2. יצירת משלוחים מרובה > הגדר ברירות מחדל
3. צור הכל
- כושל מציג שגיאה, השאר ממשיכים
- אזהרה: לא ניתן לבטל. ודאו לפני אישור.

משלוח ידני (ללא הזמנה):
1. צור משלוח / משלוחים > משלוח ידני חדש
2. מלא ידנית הכל
3. צור משלוח
שימושי להחזרות, העברות פנימיות, משלוחים מותאמים.

הבנת השדות:
- חברת משלוחים (רישיון): לפי מהירות/מיקום/עלות
- סוג משלוח: רגיל / מהיר / נקודת איסוף / לוקר
- הערות: לשליח (על המדבקה) / פנימיות
- חבילות: מספר, משקל ק"ג, מידות ס"מ (אופציונלי)

איך הזמנות Wix מתעדכנות:
1. ShipOS יוצר משלוח
2. שולח לחברה
3. חברה מחזירה מספר מעקב
4. ShipOS מעדכן Wix:
   - סטטוס -> מולא

============================================================
סקירת API
============================================================

כתובת בסיס: https://app.shipos.co.il/api/

אימות:
1. מבוסס רישיון (תוסף): license_key + domain + token
2. Sanctum (לוח בקרה): Laravel Sanctum Bearer (לא לאינטגרציות חיצוניות)

פורמט בקשה: form data / multipart לבקשות POST. פרמטרים נפוצים: license_key, domain, token, version.

תגובות JSON:
הצלחה: { "success": true, "data": {...} }
שגיאה: { "success": false, "message": "...", "errors": {...} }

קודי HTTP: 200, 201, 400, 401, 403, 404, 422, 429, 500.

מגבלות:
- 60 בקשות/דקה לרישיון
- 120 בקשות/דקה ל-IP
- 10 בקשות/דקה לפעולות מרובות
תגובה ל-429: retry_after שניות. השתמשו ב-exponential backoff.

גרסאות: version=3.1.1 (השתמשו בעדכנית).

נהגי עבודה:
1. בדקו success לפני data
2. עברו על errors לפי שדה
3. נסיון חוזר ל-5xx עם backoff
4. תיעוד שגיאות
5. אל תחשפו טוקנים בקוד לקוח/repos


============================================================
נקודות קצה משלוחים
============================================================

POST /shipping/open_new_order — יצירת משלוח חדש
חובה: license_key, domain, token, version
פרמטרי הזמנה: order_id, shipping_first_name, shipping_last_name, shipping_address_1, shipping_city, shipping_phone (חובה); shipping_address_2, shipping_postcode, shipping_email, shipping_company, total_weight, num_of_packages, delivery_type, pickup_point_id, notes, cod_amount, items (אופציונלי).

פורמט items:
  items[0][name]=Product Name
  items[0][quantity]=2
  items[0][sku]=SKU-001
  items[0][price]=99.90
  items[0][is_virtual]=false

תגובת הצלחה: order_id, shipping_code, tracking_number, tracking_url, status, label_url.

אזהרה: is_virtual חשוב. פריטים וירטואליים לא דורשים משלוח פיזי.

POST /shipping/get_order_details — פרטי משלוח/הזמנה
POST /shipping/get_shipping_status — סטטוס + היסטוריה
POST /shipping/label — מדבקה PDF (label_type אופציונלי: thermal/a4)
POST /shipping/bulk-label — מדבקות מרובות ב-PDF אחד (order_ids[])
POST /shipping/order — כל המשלוחים להזמנה (יכולים להיות מספר)
POST /shipping/change_order_status — שינוי סטטוס (למשל ביטול)

נקודות קצה מיקומי איסוף
============================================================

POST /shipping/pickup_locations
חובה: license_key, domain, token

תגובת הצלחה כוללת locations[]: id, name, address, city, postcode, phone, type, latitude, longitude, operating_hours, is_active. וגם total + provider.

סוגי מיקום: store, locker, post_office, collection_point.

סינון בצד הלקוח:
לפי עיר:
  const telAviv = res.data.locations.filter(l => l.city === "Tel Aviv");
לפי סוג:
  const lockers = res.data.locations.filter(l => l.type === "locker");
לפי קרבה (haversine):
  function getDistance(lat1, lon1, lat2, lon2) {
    const R = 6371;
    const dLat = (lat2-lat1)*Math.PI/180, dLon = (lon2-lon1)*Math.PI/180;
    const a = Math.sin(dLat/2)**2 +
              Math.cos(lat1*Math.PI/180)*Math.cos(lat2*Math.PI/180) *
              Math.sin(dLon/2)**2;
    return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
  }

מטמון: מיקומים לא משתנים תכופות, שמרו 24 שעות, רענון יומי או בסנכרון ידני.

זרימת קופה: בטעינת דף משוך מיקומים, הצג מפה/רשימה, שמור id, ביצירת הזמנה העבר pickup_point_id + delivery_type=pickup.

============================================================
Webhooks
============================================================

Webhooks מודיעים ל-ShipOS כשאירועים מתרחשים בחנות WooCommerce. הסנכרון האוטומטי משתמש ב-3 אירועים:
- order_created
- order_updated
- order_deleted

תבנית URL: https://app.shipos.co.il/api/wp/webhooks/{wp_site_id}/{event}

שדה is_virtual: התוסף מסיף את השדה לפריטים. מציין אם פיזי או דיגיטלי. אם הכל וירטואלי -> אין משלוח. ודאו שתוסף WC עדיין פעיל.

Payload בפורמט WC REST API Order schema (פרטי לקוח, פריטים, משלוח, מטא). אירועי delete מקבלים payload מינימלי (רק order ID).

Retries (WC מובנה): עד 5 נסיונות עם עיכובים 1m, 5m, 30m, 1h, 6h. אחרי 5 כשלונות מושבתים, צריך הפעלה ידנית.

אבטחה: HMAC-SHA256 בכותרת X-WC-Webhook-Signature.

חיבור:
1. ניהול WP > ShipOS > הגדרות > רישיונות
2. "חבר חברת משלוחים חדשה"
3. בחר את החברה
4. הזן קוד + כתובת + שם ידידותי
5. שמור

תכונות אופייניות (משתנות בין חברות):
- משלוח רגיל: כן
- משלוח כפול: כן (רוב)
- נקודות איסוף: כן (רוב)
- גוביינא: כן (רוב)
- אקספרס: לא
- מדבקות: כן

טיפים:
- כתובת איסוף מדויקת
- מספר מחסנים = חיבורים נפרדים
- ניתן לחבר מספר חברות Baldar במקביל

פתרון בעיות:
- "קוד חיבור לא תקין": ללא רווחים מיותרים
- חברה לא ברשימה: אולי בשם אחר, פנו לתמיכה
- חיבור נתקע: רענן ונסה שוב
- נקודות איסוף לא בקופה: הפעלה באזורי משלוח של WC
